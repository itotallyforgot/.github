# Security Policy

## Reporting a vulnerability

Report security issues privately through GitHub's
[private vulnerability reporting](https://docs.github.com/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
on the affected repository (Security tab → Report a vulnerability), or open a
draft security advisory.

Please do not open a public issue for anything exploitable. Expect an initial
response within a few business days.

## Scope

This repository holds the fleet's shared CI and security workflows. A flaw in a
reusable workflow here can affect every repo that consumes it, so workflow
reports are treated as high priority.
